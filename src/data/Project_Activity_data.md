## Project page name : Chao-dom
Page Variant : ux-case-study
Type Work : UX UI
Project name : Chao-dom (EasyDom)
Description : Term Project วิชา Human Interface Design คณะ IT สจล. หัวข้อ Easy Dorm: How to help freshy effectively and efficiently find rooms in university dormitory or private residence โดยทำงานครบ 5 phases คือ Data Gathering, User Research, Analysis, Conceptual Design และ Detailed Design & Evaluation
Preview Image : /images/Project/Preview.chaodom.png
Hero Focus : โปรเจกต์นี้ควรถูกเล่าแบบ UX Case Study เต็มรูปแบบ เน้นปัญหาจริงจากผู้ใช้ งานวิจัย การวิเคราะห์ และการทดสอบ usability มากกว่าการโชว์หน้าจอสวยอย่างเดียว
Role Responsibilities : Head Figma Design, Help Interview, Usability Testing Interview, Make Document
Tech Stack : Figma, FigJam, Microsoft Word, Canva, Adobe Premiere Pro, Microsoft Teams Recording, Gemini AI, NotebookLM
Timeline : 1 month 1 week (5 phases)
Team Members : 4 members (นายชยุต บุญวัฒน์ 67070032, นายธนัทภัทร พรหมทอง 67070069, นางสาววรรณณิศา อมรวงศ์ไพบูลย์ 67070155, นายอสิธารา พุ่มดอกไม้ 67070199)
Challenges : ประสานงานนัดสัมภาษณ์ข้ามคณะในเวลาจำกัด, บริหาร 5 เฟสในเวลา 1 เดือน 1 สัปดาห์, แปลง pain points จากผู้ใช้ 3 คนเป็น requirements/use cases, ทำ prototype ให้ละเอียดพอสำหรับ usability testing, และจัดการเอกสารหลายรูปแบบพร้อมกัน
Ideation : เกิดจากเรื่องจริงของนักศึกษาที่ลืมจ่ายค่าหอในจนหลุดจองหลายรอบ ต้องเสียเวลาหาหอนอกผ่านข้อมูลที่ล้าสมัย จึงตั้งสมมติฐานว่าแอปที่มีสถานะห้องว่างแบบ real-time, ข้อมูลขนส่ง, รีวิว, และ verified contact จะช่วยลดเวลาและความยุ่งยากได้
Overview : EasyDom เป็น prototype แอป iOS สำหรับช่วยนักศึกษาใหม่ สจล. หาหอพักได้ง่ายขึ้น โดยแก้ pain point เรื่องข้อมูลไม่อัปเดต สถานะห้องว่างไม่ชัด และการติดต่อเจ้าของหอที่ยุ่งยาก ผ่านกระบวนการ UX/UI ครบตั้งแต่งานวิจัยจนถึง usability testing
The Problem : นักศึกษาใหม่ต้องหาหอจากหลายแหล่งข้อมูลที่ไม่ตรงกับความจริง ทั้งราคา รูปภาพ เบอร์ติดต่อ สถานะห้อง และข้อมูลการเดินทาง ส่งผลให้ต้องลงพื้นที่เอง ใช้เวลาหอพักตั้งแต่ 1 วันจนถึง 3 เดือนกว่าจะตัดสินใจได้
Key Features : สถานะห้องว่าง real-time, advanced search filters, interactive transport map, side-by-side comparison, wishlist พร้อมแจ้งเตือน, verified owner contact, review จากผู้พักจริง, customer support chat
The Process :
- Phase 1 Data Gathering: เก็บเรื่องเล่าจริงจากผู้ใช้, รวบรวม pain points 7 ข้อ, ศึกษา RentHub / Facebook / Google Maps และตั้งสมมติฐาน 8 ข้อ
- Phase 2 User and Task Analysis: สัมภาษณ์ผู้ใช้ 3 คน, สังเกตพฤติกรรมค้นหาหอ, เก็บ evidence จากแพลตฟอร์มจริงและสถานที่จริง
- Phase 3 Analysis and Conceptual Design: สร้าง persona, experience map, pain/gain analysis, จัดลำดับ requirement
- Phase 4 Detailed Design: สร้าง task list, use cases, task objects, window diagram, container specification และ prototype หลายหน้าจอ
- Phase 5 Evaluation and Prototype: ทดสอบกับผู้ใช้ 3 คนด้วย thinking aloud และ post-session discussion เพื่อหาจุดปรับปรุงจริง
The Solution : iOS prototype ที่ทำให้การหาหอพักโปร่งใสขึ้น เห็นข้อมูลสำคัญในที่เดียว และตัดสินใจได้บนข้อมูลที่เชื่อถือได้มากกว่าเดิม
Impact : มีงานวิจัยผู้ใช้จริง 3 คน, prototype มากกว่า 10 หน้าจอ, UI components มากกว่า 25 รายการ, และพบ pain points หลังทดสอบจริง 15+ จุดเพื่อใช้ปรับปรุงต่อ

## Project page name : Multimedia Learning Polygon Mesh
Page Variant : interactive-learning-case
Type Work : UX/UI (Interactive Media)
Project name : Multimedia Learning Polygon Mesh (Topic: 19 Polygon Mesh)
Description : สื่อการเรียนรู้แบบ interactive สำหรับอธิบายการสร้าง polygon mesh และ 3D object representation ผ่าน flow 2D -> 3D -> polygon study -> jigsaw game
Preview Image : /images/Project/preview-polygon-mesh.png
Hero Focus : ควรเล่าเป็นโปรเจกต์สื่อการเรียนรู้ ไม่ใช่แค่หน้า UI สวย ๆ โดยเน้นวิธีทำให้หัวข้อเทคนิคที่เข้าใจยากกลายเป็น interactive learning experience
Role Responsibilities : Head Figma Design / Interactive Flow Design
Tech Stack : Figma, Figma Prototype, Figma Site, Pen Tool, Figma Plugins (Image Cutter, Property Randomizer)
Timeline : 1 month
Team Members : 3 (จตุรภัทร กิติมาโภคิน, ณัฐวุฒิ ทิพย์รัตน์, ธนัทภัทร พรหมทอง)
Challenges : ทำให้เรื่อง polygon mesh และ 3D object representation ซึ่งค่อนข้าง abstract เข้าใจง่ายขึ้นผ่าน interaction, animation, hover states, draggable pieces และ content flow ที่ผู้เรียนกดสำรวจได้เอง
Ideation : ใช้แนวคิด game-based learning และ interactive explanation เพื่อเปลี่ยนเนื้อหาทฤษฎีให้เป็นประสบการณ์เรียนรู้แบบลงมือสำรวจทีละขั้น
Overview : โปรเจกต์นี้เป็นสื่อการเรียนรู้ที่พาผู้ใช้เข้าใจตั้งแต่ vertices, edges, faces ไปจนถึงการเกิด polygon mesh และ 3D object representation พร้อมต่อยอดด้วย jigsaw game ที่ให้ผู้เรียนประกอบ face ของโมเดลด้วยตัวเอง
The Problem : หัวข้อ polygon mesh ยากต่อการจินตนาการจาก text-only หรือ slide แบบนิ่ง จึงจำเป็นต้องมีสื่อที่แสดงลำดับการเกิดรูปทรงอย่างเป็นภาพและมี interaction ช่วยให้ผู้เรียนเชื่อมโยงแนวคิดได้ง่ายขึ้น
Key Features : 2D Process Polygon, 2D Modify/Config, 3D Process Polygon, 3D Config, Polygon Study, 3D Object Representation, Finalizing 3D Object Modeling, Creating Polygon Mesh แบบ hover explanation, Jigsaw Polygonal และ Jigsaw Polygonal Game แบบ draggable real-time
The Process :
- ออกแบบ home และ topic navigation ให้ผู้เรียนเลือกเส้นทางการเรียนได้ง่าย
- แยกเนื้อหา 2D และ 3D พร้อมลูกศร next/back และ dropdown config เพื่อควบคุมการเรียนแบบ step-by-step
- ใช้ hover / prototype linking / while hovering เพื่อทำให้เนื้อหานิ่ง ๆ มีความเคลื่อนไหวและตอบสนอง
- สร้าง jigsaw game ด้วย Pen Tool, Image Cutter และ Property Randomizer เพื่อสุ่มชิ้นส่วนและให้ลากประกอบได้จริงบน Figma Site
The Solution : สื่อการเรียนรู้ที่ผู้ใช้สามารถเห็นการเปลี่ยนแปลงของ object ทีละส่วน กดสำรวจหัวข้อเชิงลึก และลงมือเล่นผ่าน jigsaw เพื่อ reinforce ความเข้าใจ
Impact : ช่วยเปลี่ยนเนื้อหาพื้นฐาน 3D modeling ให้เข้าใจง่าย สนุก และจดจำได้ดีขึ้นผ่านการมีส่วนร่วมของผู้เรียน

## Project page name : VPS-Tycoon
Page Variant : systems-game-case
Type Work : programming game dev
Project name : VPS TYCOON
Description : เกมจำลองธุรกิจบริการเช่าเซิร์ฟเวอร์ (VPS hosting) ที่ผู้เล่นรับบทเป็นเจ้าของบริษัท ต้องบริหาร rack, virtual machines, ลูกค้า, เวลาในเกม และ event ต่าง ๆ ผ่าน game loop เชิงระบบ
Preview Image : /images/Project/preview-vps-tycoon.png
Hero Focus : ควรเล่าแบบ programming + systems design case study เน้น core loop, game systems, logic และธีม cyberpunk มากกว่าการบอกว่าเป็นเกมธรรมดา
Role Responsibilities : Developer / Programmer (ควรยืนยันบทบาทย่อยเพิ่มเติมก่อนนำขึ้นหน้าเว็บ)
Tech Stack : Java, JavaFX, IntelliJ IDEA, GitHub, Aseprite, Canva, Discord
Timeline : 1 ภาคเรียน (ภาคเรียนที่ 1 ปีการศึกษา 2568)
Team Members : LoveJarnBank Group (จำนวนสมาชิกควรใส่ตามทีมจริงอีกครั้งก่อนขึ้นหน้าเว็บ)
Challenges : จำลองระบบธุรกิจ VPS ที่มีทรัพยากรหลายชั้นให้เล่นสนุกและเข้าใจง่าย, ออกแบบความสัมพันธ์ระหว่าง rack / VM / requests / skill / events, และทำ UI ธีม cyberpunk ให้เข้ากับเนื้อหาเชิงเทคนิค
Ideation : เปลี่ยนแนวคิดเรื่อง server management และ VPS hosting ซึ่งดู technical มาก ให้กลายเป็นเกมบริหารธุรกิจที่มีความก้าวหน้า มีการตัดสินใจ และมีระบบอัปเกรดชัดเจน
Overview : ผู้เล่นเริ่มต้นในปี 2000 บริหารบริษัท VPS hosting ด้วยทรัพยากรหลักคือ money และ rating ซื้อ server ติดตั้งบน rack รับ requests จากลูกค้าผ่าน messenger ปรับสเปก VM จัดการเวลา deploy รับมือ event ระหว่างเช่า และอัปเกรด skill 6 ด้านเพื่อขยายกิจการ
The Problem : การนำหัวข้อเทคนิคอย่าง VPS, rack, virtual machine, network และ security มาทำให้เข้าใจง่ายพอสำหรับผู้เล่นทั่วไปโดยยังคงความลึกของระบบ
Key Features : Rack management, VM specification matching, Messenger request system, time simulation (30 วินาทีจริง = 1 วันในเกม), 6 skill trees (Deploy, Networks, Security, Marketing, Management, Rack Slot), event system, save/load system, rating progression
The Process :
- วาง core concept และ game loop ให้ผู้เล่นรับ request -> จัดสรร VM -> deploy -> รับรายได้ -> อัปเกรดระบบ
- ออกแบบ data relationship ของ company, requests, rack configuration และ VM assignments
- พัฒนาระบบเวลาและ event system เพื่อสร้างแรงกดดันและ progression
- สร้าง UI ธีม cyberpunk และใช้ pixel art/graphic support เพื่อให้เกมมีเอกลักษณ์
The Solution : เกมจำลองธุรกิจเชิงระบบที่มีทั้งความรู้สึกของ tycoon game และการจัดการ resource เชิงเทคนิค พร้อมระบบ save/load และ event-driven progression
Impact : ได้ฝึก OOP, game system design, state management, และการทำงานร่วมกันในโปรเจกต์ขนาดทีม

## Project page name : Drive@KMITL
Page Variant : fullstack-product-case
Type Work : programming
Project name : DriveChat@kmitl
Description : เว็บไซต์แชทสำหรับพูดคุยกับคนแปลกหน้าแบบสุ่ม โดยเปลี่ยนบรรยากาศตามสถานที่ต่าง ๆ ในมหาวิทยาลัยเสมือนกำลังเดินทางด้วยพาหนะ
Preview Image : /images/Project/preview-drivechat.png
Hero Focus : ควรเล่าแบบ product + full-stack case study เน้น problem, product concept, real-time architecture, room flow, limitations และสิ่งที่ทำได้จริง
Role Responsibilities : Backend (50%), Frontend (5%), Document (40%)
Tech Stack : Next.js, FastAPI, WebSockets, Tailwind CSS, Figma, GitHub Desktop
Timeline : 1 ภาคเรียน (ภาคเรียนที่ 1 ปีการศึกษา 2567)
Team Members : 5 (ธนัทภัทร พรหมทอง, วรากร ตั้งเจริญอารีย์, วราภรณ์ สุขม่วง, วรศิรา เสนาชัยบาล, อธิการ ชินจิรัฐติกาล)
Challenges : ทำระบบ real-time communication ให้เปลี่ยนห้องอัตโนมัติได้โดยไม่หลุด connection, เชื่อม Backend กับ Frontend ให้เสถียร, และจัดการ room lifecycle เมื่อมีผู้ใช้เข้าออกหลายบทบาท
Ideation : สร้างพื้นที่ที่คนสามารถคุยหรือระบายกับคนแปลกหน้าได้อย่างสบายใจ พร้อมบรรยากาศการเดินทางในมหาวิทยาลัยที่เปลี่ยนไปเรื่อย ๆ เพื่อไม่ให้การคุยน่าเบื่อ
Overview : เว็บไซต์นี้แบ่งผู้ใช้เป็น Driver และ Passenger มีระบบสร้างห้อง, สุ่มเข้าห้อง, จำกัดจำนวนคนตามประเภทรถ, countdown ก่อนย้ายห้อง, พื้นหลังวิดีโอแต่ละสถานที่ และบรรยากาศการคุยที่เปลี่ยนไปตาม room type
The Problem : ผู้ใช้หลายคนอยากมีพื้นที่ปลอดภัยสำหรับคุยกับคนไม่รู้จักโดยไม่ถูกตัดสิน แต่แพลตฟอร์มทั่วไปไม่ได้ออกแบบประสบการณ์ random chat แบบมีธีมและ room transition ชัดเจน
Key Features : Username setup, Passenger / Driver roles, random join, create room, room type selection, room capacity control, countdown to next room, live room status, location-based video backgrounds, auto remove room when no users remain
Room Types : Bicycle (2 คน), Taxi (4 คน), Songthaew (10 คน), EV / Minibus (15 คน)
The Process :
- ออกแบบ flow หน้าหลัก, join room, create room และ chat room ใน Figma
- พัฒนา backend API และ websocket communication ระหว่าง frontend กับ backend
- สร้างระบบ create room, random join, role-based access และ room type restrictions
- ปรับปรุง room status, user count, room transitions และหน้าเว็บให้สมบูรณ์ขึ้น
The Solution : แพลตฟอร์ม random chat ที่ผสมแนวคิด journey experience เข้ากับ real-time web app ทำให้การคุยกับคนแปลกหน้ามีทั้งบริบทและความแปลกใหม่
