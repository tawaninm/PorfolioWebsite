## Project page name : Chao-dom

Type Work : UX UI
Project name : Chao-dom (EasyDom)
Description : Term Project วิชา Human Interface Design คณะ IT สจล. หัวข้อ Easy Dorm: How to help freshy effectively and efficiently find rooms in university dormitory or private residence แบ่ง 5 เฟส: Data Gathering, User Research, Analysis, Conceptual Design, Detailed Design & Evaluation
Role Responsibilities : Head Figma Design, Help Interview, Usability Testing Interview, Make Document
Tech Stack : Figma, FigJam, Microsoft Word, Canva, Adobe Premiere Pro, Microsoft Teams Recording, Gemini AI, NotebookLM
Timeline : 1 month 1 week (5 phases)
Team Members : 4 members (นายชยุต บุญวัฒน์ 67070032, นายธนัทภัทร พรหมทอง 67070069, นางสาววรรณณิศา อมรวงศ์ไพบูลย์ 67070155, นายอสิธารา พุ่มดอกไม้ 67070199)

Challenges : ประสานงานนัดสัมภาษณ์ข้ามคณะในเวลาจำกัด บริหาร 5 เฟสใน 1 เดือน 1 สัปดาห์ แปลง Pain Points จากผู้ใช้ 3 คนเป็น Requirements/Use Cases ทำ Prototype ให้ละเอียดพอสำหรับ Usability Testing จัดการเอกสารหลายรูปแบบพร้อมกัน

Ideation : เกิดจากเรื่องจริงนักศึกษาลืมจ่ายค่าหอในหลุดจองต้องเดินหาหอนอกหลายเดือนผ่านข้อมูลล้าสมัย ตั้งสมมติฐานว่าแอป Real-time มีสถานะห้องว่างยืนยัน ข้อมูลขนส่งสาธารณะ ติดต่อเจ้าของหอ Verified จะลดเวลาและความยุ่งยาก ยืนยันจากสัมภาษณ์ 3 คน

Overview : EasyDom (Chao-dom) เป็น Term Project วิชา Human Interface Design คณะ IT สจล. หัวข้อ How to help freshy find rooms ดำเนินการ UX/UI ครบ 5 เฟส ตั้งแต่เก็บข้อมูล วิจัยผู้ใช้เชิงลึก วิเคราะห์ Persona/Experience Map ออกแบบ Conceptual Design ไปจนถึง Detailed Prototype iOS ตาม Apple HIG และ Guerilla Usability Testing ผลลัพธ์คือ Prototype แอป iOS ที่แก้ปัญหาข้อมูลล้าสมัย สถานะห้องไม่ชัดเจน และการติดต่อเจ้าของหอที่ยุ่งยาก

The Problem : นักศึกษาใหม่ สจล. ประสบปัญหาหาหอพักนอกสถาบัน ข้อมูลบน RentHub Google Maps Facebook ไม่เป็นปัจจุบัน ราคา สถานะห้อง รูปภาพ เบอร์ติดต่อไม่ตรงความจริง ติดต่อแล้วถูกปฏิเสธ ต้องลงพื้นที่สำรวจเอง ระบบจองหอในไม่โปร่งใส ข้อมูลการเดินทาง ขนส่ง สภาพแวดล้อม นิสัยเจ้าของหอไม่มีแพลตฟอร์มใดให้ครบ สัมภาษณ์ 3 คน (โปเต้ ฟ้าใส ทัก) ทุกคนประสบปัญหาเดียวกัน ใช้เวลา 1 วัน ถึง 3 เดือนกว่าจะได้หอ
Pain Points 7 ข้อ: (1) ข้อมูลออนไลน์ไม่อัปเดต เบอร์โทร รูปภาพ (2) ติดต่อออนไลน์ถูกปฏิเสธ ต้องไปสำรวจจริง (3) สถานะห้องว่างไม่ชัดเจน (4) ไม่ทราบเส้นทางเดินทาง สภาพแวดล้อมรอบหอ (5) ประชาสัมพันธ์ปากต่อปาก ติดต่อลำบาก (6) ความปลอดภัยไม่คุมเข้ม (7) สถานะจองหอในไม่ชัดเจน
True Story: นางสาวภัชรธนสรณ์ พีรวัฒน์กุล ลืมจ่ายค่าหอในรอบ 1 หลุดจอง 3 รอบ หาหอซอยเกกีเต็มหมด เส้นลาดกระบังโทรถูกปฏิเสธ เบอร์ไม่ตรง ต้องเดินสำรวจเอง สุดท้ายได้หอแถวหัวตะเข้ เส้นหลักรถเยอะเสียงดัง

## Project page name : Multimedia Learning Polygon Mesh

**Type Work** : UX/UI (Interactive Media)
**Project name** : Multimedia Learning Polygon Mesh (Topic: 19 Polygon Mesh)
**Description** : สื่อการเรียนรู้แบบ Interactive เกี่ยวกับการสร้างวัตถุ 3 มิติ (3D Object Representation) โดยเน้นกระบวนการเกิด Polygon Mesh ตั้งแต่การสร้าง Vertices, Edges ไปจนถึง Face รวมถึงการเรียนรู้ผ่านมินิเกม Jigsaw
**Role Responsibilities** : Head figma design (รับผิดชอบการออกแบบโครงสร้าง UI/UX และระบบ Interactive ทั้งหมดบน Figma)
**Tech Stack** : Figma (Interactive Prototype, Animation, Figma Plugins: Image Cutter, Property Randomizer)
**Timeline** : 1 month
**Team Members** : 3 (จตุรภัทร กิติมาโภคิน, ณัฐวุฒิ ทิพย์รัตน์, ธนัทภัทร พรหมทอง)
**Challenges** : การออกแบบเนื้อหาทางเทคนิคที่ซับซ้อน (เช่น พิกัด Vertices) ให้เข้าใจง่ายผ่านสื่อ Interactive และการใช้เทคนิคขั้นสูงใน Figma เพื่อสร้างระบบเกม Jigsaw ที่สามารถสุ่มชิ้นส่วนได้แบบ Real-time
**Ideation** : ใช้แนวคิด "Game-based Learning" เพื่อลดความน่าเบื่อของเนื้อหาทฤษฎี โดยให้ผู้เรียนมีปฏิสัมพันธ์กับสื่อในทุกขั้นตอนของการสร้างโมเดล 3D
**banner image** : `[path/to/Polygon_Mesh_Banner.png]` (ภาพปกจากหน้าแรกของไฟล์ 19 Creating polygon mesh\_Sec1.pdf)
**Overview** : สื่อการสอนที่ครอบคลุมเนื้อหา 2D/3D Process Polygon, Polygonal Study และ Jigsaw Polygonal Game
**The Problem** : เนื้อหาเรื่อง Polygon Mesh เป็นพื้นฐานที่ยากต่อการจินตนาการหากไม่มีสื่อประกอบที่แสดงลำดับขั้นการสร้างที่ชัดเจน
**The Process** : วางแผนเนื้อหา -\> ออกแบบ UI ใน Figma -\> สร้างระบบ Interactive (Hover, Animation) -\> พัฒนา Jigsaw Game โดยใช้ Pentool และ Plugin ในการสุ่มตำแหน่งชิ้นส่วน
**Gallery image 1-4**: `[path/to/Image_1.png]` (หน้า Home), `[path/to/Image_2.png]` (2D Process), `[path/to/Image_3.png]` (3D Process), `[path/to/Image_4.png]` (Jigsaw Game)
**The Solution** : สร้างสื่อที่ผู้ใช้สามารถกดดูความเปลี่ยนแปลงของวัตถุได้ทีละส่วน และทดลองประกอบชิ้นส่วนด้วยตัวเองผ่านหน้าจอ Draggable Jigsaw
**Impact** : ช่วยให้นักศึกษาเข้าใจโครงสร้างพื้นฐานของโมเดล 3D ได้รวดเร็วและสนุกสนานขึ้น
